# Planning Intelligent – Comment présenter au DG

## 1. Une seule page, tout en un
Ouvrir **Menu → Planning Intelligent**. La page affiche d’un coup : le **cockpit** (mois, DRANEF/DPANEF, ADP), le **score Impact Planning** (0–100) et le **risque opérationnel**, les **4 cartes KPI** (couverture territoriale, animation & médiation, PDFCP/PDFC, logistique & coût), les **Top 5 priorités** du mois, le **planning** (liste ou calendrier), l’**Assistant Décisionnel** (alertes + recommandations avec « Pourquoi ») et le **Dossier DG** en 6 rubriques. Présenter comme un « tableau de bord décisionnel » : tout ce dont le DG a besoin pour arbitrer et valider est sur un seul écran.

## 2. Données fictives « démo DG »
En l’absence de planning pour le mois courant, la page tourne en **mode démo** (badge « Mode démo DG ») : les KPIs, le score, les priorités, les alertes, les recommandations et le Dossier DG sont pré-remplis avec des exemples crédibles. Objectif : montrer la **valeur** de l’outil dès la première ouverture, sans dépendre de la saisie terrain. Insister sur le fait que, une fois les plannings renseignés par les ADP/DPANEF, les mêmes blocs afficheront les **vrais** chiffres et recommandations.

## 3. Actions concrètes pour le DG
Montrer les trois actions immédiates : **« Générer note DG (PDF/Print) »** (impression ou sauvegarde PDF pour archivage / comité), **« Copier en format note »** (coller le Dossier DG dans un mail ou une note pour diffusion), et la section **« Décisions attendues »** en fin de Dossier DG (DPANEF / DRANEF / DG). Résumer : « Le DG voit en une page le contexte, les impacts, les risques, les besoins logistiques et les décisions à prendre. »
